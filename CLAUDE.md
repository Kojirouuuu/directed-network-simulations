## Project Overview

有向ネットワーク上での SAR（Susceptible-Adopted-Recovered）モデルによる情報拡散・感染症伝播シミュレーション。Java メインプロジェクトと、C 言語の EBCM（Edge-Based Compartmental Model）数値解析、Python の EBCM 解析コンポーネントで構成される。

## Commands

### Java (Gradle)

```bash
./gradlew build           # Build the project
./gradlew test            # Run all tests
./gradlew :app:runSAR     # Run SAR simulation (main entry point)
./gradlew :app:runGraphGen  # Generate graphs and export edge lists
./gradlew run             # Run sim.App (demo/verification)
```

### C (EBCM)

```bash
# Compile and run (see scripts/run-c.sh)
clang -fopenmp -std=c11 -Wall -Wextra -Wpedantic -O0 -g c-lang/ebcm/same-in-out.c -o build/same-in-out
./build/same-in-out
```

Build output goes to `build/` (created automatically).

### Scripts

```bash
./scripts/run-sar.sh       # Build + run SAR
./scripts/run-graphgen.sh  # Build + run GraphGen
./scripts/run-c.sh         # Compile + run C EBCM
```

## Architecture

### Java module (`app/src/main/java/sim/`)

**Entry points:**
- `SAR.java` — Parallel batch SAR simulation. Configure via the inner `SimulationConfig` class (network type, N, lambda, threshold, batch size, etc.). Uses `ForkJoinPool` for parallelism.
- `GraphGen.java` — Generates graphs and writes NetworkX-compatible edge lists to `out/edgelist/`.
- `App.java` — Demo/verification entry point.

**Core graph (`network/`):**
- `DirectedGraph.java` — CSR (Compressed Sparse Row) graph supporting mixed directed and undirected edges. Undirected edges are stored internally as two directed arcs with an `isUndirected` flag. Key methods: `outNeighborRange`, `inNeighborRange`, `fromEdgeListWithUndirectedFlag`, `loadFromEdgeList`, `reverseDirectedEdges`, `writeEdgeList`, `printInfo`, `checkConnected`.
- `network/topology/` — Graph generators: `DirectedCMOutPow`, `DirectedCMInPow`, `DirectedCM`, `PowPow`, `SameInOut`, `DirectedBA`, `EgoTwitter`, `Gplus`, `HiggsSocial` (and their `Rev*` variants). Undirected: `ER`, `BA`, `CM`.

**Simulation (`simulation/`):**
- `SARSimulator.java` — Event-driven SAR simulator (non-Gillespie).
- `SARGillespieSimulator.java` — Gillespie-algorithm SAR simulator.
- `SARResult.java` — Holds time-series (S, A, R, Phi) and provides `writeTimeSeriesCsv` / `writeFinalStateCsv`.

**Utilities (`utils/`):**
- `SwitchUtils.java` — Central dispatch for graph generation and output path construction. **Add new network types here** (both `generateGraph` and `buildNetworkPath` switch statements).
- `ArrayUtils.java`, `RandomUtils.java`, `PathsEx.java` — General helpers.

### Adding a new network type

1. Create `sim/network/topology/MyNetwork.java` with a static `generate(...)` method.
2. Add a case to `SwitchUtils.generateGraph` and `SwitchUtils.buildNetworkPath`.
3. Update `SimulationConfig.networkType` in `SAR.java` to use it.

### Output directory structure

```
out/sar/{optionPath}/threshold={T}/{networkType}/N={N}/{networkSpecific}/results_{batchIdx}.csv
out/edgelist/{networkPath}/{batchIndex}.csv   # for loadFromEdgeList=true
```

### C EBCM (`c-lang/ebcm/`)

Three C programs solving the EBCM ODEs analytically/numerically for different degree distribution assumptions:
- `directed-infty.c` — Directed network with infinite max-degree
- `same-in-out.c` — Directed network where in-degree = out-degree (power-law); OpenMP parallel
- `undirected-infty.c` — Undirected network with infinite max-degree

### Python EBCM (`notebooks/utils/`)

Python module used from Jupyter notebooks:
- `directed_ebcm.py` — Directed EBCM computation
- `ebcm.py` — General EBCM
- `calculation.py`, `switch.py` — Helpers

Main notebook: `notebooks/directed-ebcm.ipynb`

## Key Parameters in `SAR.SimulationConfig`

| Field | Description |
|---|---|
| `networkType` | String key dispatched by `SwitchUtils` |
| `loadFromEdgeList` | If `true`, loads graphs from `out/edgelist/` (generated by GraphGen) |
| `isFinal` | Output only final state (`writeFinalStateCsv`) vs full time series |
| `batchSize` | Number of independent graph realizations run in parallel |
| `itrs` | Iterations per batch index |
| `lambdaDirectedList` | Sweep range for directed-edge infection rate |
| `lambdaNondirectedList` | Sweep range for undirected-edge infection rate |
| `threshold` | SAR adoption threshold (S→A requires this many cumulative infections) |
